# Fast ISTQB CTFL v4.0 Preparation With This Repo

This guide is a focused plan for using this repository to prepare for the ISTQB Certified Tester Foundation Level (CTFL) v4.0 exam.

It does not guarantee a pass. Use it together with the official syllabus, official glossary, and official sample exams.

## Before you book the exam

Book the exam only when you can consistently:

- explain the six CTFL v4.0 syllabus chapters in your own words
- score at least 30/40 on practice exams more than once
- explain why wrong answers are wrong
- solve technique questions without memorizing question wording
- finish a 40-question practice session within the exam time limit

## Step 1: Learn the exam shape

Start in the app dashboard and review:

- 40 questions
- 60 minutes
- 26/40 passing score
- six syllabus chapters

Then open the official resources linked in the app:

- ISTQB CTFL v4.0 overview
- ISTQB CTFL v4.0 syllabus downloads
- ISTQB glossary
- official sample exams A, B, C, and D

Use the app's official syllabus accelerator as your study map. It is not a copy of the syllabus; it is an original learning path based on the official structure:

- 14 business outcomes
- 64 learning objectives
- six chapters
- chapter weights from the public exam structure

Keep the official PDF open when you want exact wording. Use this repo to turn that wording into active practice.

## Step 2: Study one chapter at a time

For each chapter:

1. Read the chapter card in the app.
2. Review the learning goals.
3. Read the key concepts.
4. Use the study tactics as mini exercises.
5. Mark the chapter as reviewed only after you can explain it without looking.

Recommended chapter order:

1. Fundamentals of Testing
2. Testing Throughout the SDLC
3. Static Testing
4. Test Analysis and Design
5. Managing the Test Activities
6. Test Tools

Use exam weight to decide how much practice time to spend:

| Chapter | Approx. exam weight | Practical implication |
| --- | ---: | --- |
| Test Analysis and Design | 27.5% | Spend extra time applying techniques, not just naming them. |
| Managing the Test Activities | 22.5% | Practice risk, monitoring, reporting, and defect-management decisions. |
| Fundamentals of Testing | 20% | Know the principles and vocabulary precisely. |
| Testing Throughout the SDLC | 12.5% | Be clear on test levels, test types, and change-related testing. |
| Static Testing | 10% | Understand review value, roles, and feedback quality. |
| Test Tools | 7.5% | Focus on benefits, risks, and adoption factors. |

## Step 3: Use active recall

For each chapter:

- open the flashcards
- answer out loud, then select **Show answer**
- grade yourself honestly: **Again**, **Good**, or **Easy**
- write down weak terms
- check weak terms in the official glossary

Grading drives the built-in spaced repetition: cards you mark **Again** come back sooner, while **Good** and **Easy** push a card up through the mastery levels until it reaches the top level and counts as mastered. Watch the per-card **Mastery level** indicator and the "mastered" read-out to see which terms still need work.

If you cannot explain a term simply, mark it **Again** and do not count it as learned.

## Step 4: Practice with the in-app quiz

Use chapter practice first:

1. Pick a chapter.
2. Answer the practice questions.
3. Submit answers.
4. Read every rationale, including correct answers.
5. Retry the weakest chapter later.

Do not memorize the questions. Focus on the reasoning.

Give extra repetition to Chapter 4 and Chapter 5. Together they are about half of the exam, and they contain many questions where two answers can look plausible.

For Chapter 4, make sure you can apply:

- equivalence partitioning
- 2-value boundary value analysis
- 3-value boundary value analysis
- decision table testing
- state transition testing
- statement and branch coverage

For Chapter 5, make sure you can reason about:

- product risk and project risk
- risk-based test prioritization
- test monitoring and test control
- test reporting and residual risk
- defect reports
- configuration management

## Step 5: Use scenario drills

Scenario drills help you connect exam theory to practical testing decisions.

For each scenario:

1. Answer as if explaining to a teammate.
2. Compare your answer with the coaching hint.
3. Add missing vocabulary from the syllabus or glossary.

This is especially useful for:

- risk-based testing
- reviews
- test levels and test types
- tool benefits and risks

## Step 6: Calibrate with official sample exams

After you finish all six chapters:

1. Take official Sample Exam A under timed conditions.
2. Review the official answer explanations.
3. Return to this app and practice weak chapters.
4. Repeat with Sample Exams B, C, and D.

Keep a simple mistake log:

```text
Date:
Sample exam:
Score:
Weak chapter:
Mistake pattern:
What I will review:
```

Common mistake patterns:

- confusing testing and debugging
- mixing test levels with test types
- missing boundary values
- choosing an answer that is true but not the best answer
- ignoring risk impact
- overestimating what test automation can prove

## Step 7: Use external resources selectively

The app links curated external resources:

- official/member-board resources
- blog explanations
- mock-test sites
- community discussions
- YouTube tutorials

Use them when a concept does not click. Do not jump between too many resources at once.

Recommended approach:

1. Start with the official syllabus.
2. Use this app for practice and recall.
3. Use one external explanation if a topic remains unclear.
4. Return to official sample exams for calibration.

## Fast review checklist

Before exam day, make sure you can answer these without notes:

- What is the difference between error, defect, and failure?
- Why can testing not prove the absence of defects?
- What are component, component integration, system, system integration, and acceptance testing?
- What is the difference between functional, non-functional, white-box, and change-related testing?
- Why are reviews useful before code execution?
- How do equivalence partitioning and boundary value analysis differ?
- When should decision table testing be used?
- What do statement coverage and branch coverage measure?
- How does product risk influence test priority?
- What makes a defect report useful?
- What are common benefits and risks of test automation?

## Final readiness rule

You are probably ready to book the exam when:

- you finish official sample exams within the time limit
- you score 30/40 or higher repeatedly
- your mistake log no longer shows the same repeated weakness
- you can explain answer rationales without memorizing wording

If you are still close to 26/40, keep practicing before booking.
